Manual Testing FAQs

Q1: What is the difference between Smoke and Sanity testing?

Smoke Testing: Performed to verify critical functionalities work. Done after build deployment. Wide and shallow coverage.

Sanity Testing: Performed to verify specific functionality after bug fixes. Narrow and deep coverage. Subset of regression testing.

Q2: Explain the Bug Life Cycle.

New → Assigned → Open → Fixed → Retest → Verified → Closed

Other states: Rejected, Deferred, Duplicate, Reopened

Q3: What is the difference between Severity and Priority?

Severity: Impact of the bug on application functionality (Critical, Major, Minor, Trivial)

Priority: Order in which bug should be fixed (High, Medium, Low)

Example: Spelling mistake on homepage - Low Severity, High Priority

Q4: What are the different types of testing?

Functional: Unit, Integration, System, UAT, Smoke, Sanity, Regression

Non-Functional: Performance, Load, Stress, Security, Usability, Compatibility

Q5: What is Regression Testing?

Testing performed to ensure new code changes haven't adversely affected existing functionality. Done after bug fixes or new feature additions.

Q6: What is Test Case and Test Scenario?

Test Scenario: High-level functionality to be tested (What to test)

Test Case: Detailed steps with expected results (How to test)

Q7: What is Exploratory Testing?

Simultaneous learning, test design, and execution without formal test cases. Tester explores the application to find defects.

Q8: What is Ad-hoc Testing?

Informal testing without documentation or planning. Done randomly to break the system.

Automation Testing FAQs

Q1: What is the difference between findElement() and findElements()?

findElement(): Returns single WebElement. Throws NoSuchElementException if not found.

findElements(): Returns List of WebElements. Returns empty list if not found.

Q2: Explain different types of waits in Selenium.

Implicit Wait: Global wait applied to all elements. driver.manage().timeouts().implicitlyWait(10, TimeUnit.SECONDS)

Explicit Wait: Wait for specific condition. WebDriverWait wait = new WebDriverWait(driver, 10)

Fluent Wait: Wait with polling interval and ignore exceptions.

Q3: How do you handle dynamic elements in Selenium?

- Use dynamic XPath with contains(), starts-with(), text()

- Use CSS selectors with partial attribute matching

- Implement explicit waits

- Use relative locators (Selenium 4)

Q4: What is Page Object Model (POM)?

Design pattern where each web page is represented as a class. Page elements are defined as variables and actions as methods. Improves maintainability and reduces code duplication.

Q5: How do you handle alerts in Selenium?

driver.switchTo().alert().accept() - Accept alert

driver.switchTo().alert().dismiss() - Dismiss alert

driver.switchTo().alert().getText() - Get alert text

driver.switchTo().alert().sendKeys("text") - Enter text in prompt

Q6: What is TestNG and its advantages?

Testing framework inspired by JUnit. Advantages: Annotations, parallel execution, data-driven testing, test prioritization, grouping, reporting, dependency management.

Q7: How do you handle frames in Selenium?

driver.switchTo().frame(index/name/WebElement)

driver.switchTo().defaultContent() - Switch back to main page

driver.switchTo().parentFrame() - Switch to parent frame

Q8: What is the difference between close() and quit()?

close(): Closes current browser window

quit(): Closes all browser windows and ends WebDriver session

API Testing FAQs

Q1: What is the difference between REST and SOAP?

REST: Architectural style, uses HTTP methods, supports JSON/XML, stateless, lightweight

SOAP: Protocol, uses only XML, has built-in security, stateful/stateless, heavier

Q2: Explain HTTP methods used in REST API.

GET: Retrieve data

POST: Create new resource

PUT: Update entire resource

PATCH: Partial update

DELETE: Delete resource

Q3: What are common HTTP status codes?

200 OK, 201 Created, 204 No Content

400 Bad Request, 401 Unauthorized, 403 Forbidden, 404 Not Found

500 Internal Server Error, 502 Bad Gateway, 503 Service Unavailable

Q4: What validations do you perform in API testing?

- Status code validation

- Response time validation

- Response body validation (JSON schema)

- Header validation

- Data type and format validation

Q5: How do you handle authentication in API testing?

Basic Auth: Username and password in header

Bearer Token: JWT token in Authorization header

OAuth 2.0: Access token obtained through OAuth flow

API Key: Key passed in header or query parameter

Q6: What is the difference between PUT and PATCH?

PUT: Replaces entire resource. All fields must be provided.

PATCH: Partial update. Only modified fields need to be sent.

Q7: How do you test API performance?

- Measure response time

- Load testing with multiple concurrent users

- Stress testing to find breaking point

- Monitor server resources (CPU, memory)

Q8: What is JSON Schema validation?

Validates JSON response structure, data types, required fields, and constraints. Ensures API contract is maintained.

Database Testing FAQs

Q1: What is the difference between DELETE and TRUNCATE?

DELETE: DML command, can use WHERE clause, slower, can be rolled back, triggers fire

TRUNCATE: DDL command, removes all rows, faster, cannot be rolled back, triggers don't fire

Q2: Explain different types of JOINs.

INNER JOIN: Returns matching records from both tables

LEFT JOIN: All records from left table and matching from right

RIGHT JOIN: All records from right table and matching from left

FULL OUTER JOIN: All records from both tables

Q3: What is the difference between WHERE and HAVING?

WHERE: Filters rows before grouping, cannot use aggregate functions

HAVING: Filters groups after GROUP BY, can use aggregate functions

Q4: What are ACID properties?

Atomicity: All or nothing transaction

Consistency: Database remains in valid state

Isolation: Concurrent transactions don't interfere

Durability: Committed data is permanent

Q5: What is the difference between Primary Key and Unique Key?

Primary Key: Cannot be NULL, only one per table, creates clustered index

Unique Key: Can have one NULL, multiple per table, creates non-clustered index

Q6: What is Normalization and its types?

Process of organizing data to reduce redundancy.

1NF: Atomic values, no repeating groups

2NF: 1NF + No partial dependency

3NF: 2NF + No transitive dependency

Q7: What is the difference between Stored Procedure and Function?

Stored Procedure: Can return multiple values, can use DML, cannot be called in SELECT

Function: Returns single value, cannot use DML, can be called in SELECT

Q8: What validations do you perform in Database testing?

- Data integrity validation

- Schema validation

- CRUD operations validation

- Stored procedures and triggers testing

- Performance testing (query optimization)
